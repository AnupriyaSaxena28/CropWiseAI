# Deploying CropWise AI on Render

Two independent services + a database:

| Service | What | Render type |
|---|---|---|
| `cropwise-db` | PostgreSQL (backend data) | Managed Postgres |
| `cropwise-backend` | Spring Boot REST API (used by the Android app) | Docker web service |
| `cropwise-web` | Next.js website (self-contained + Firebase) | Node web service |

The website and backend are **independent** — the site talks to Firebase and its
own API routes, not to the Spring Boot service.

---

## Option A — one-click Blueprint (recommended)

1. Commit and push the new files to GitHub:
   ```bash
   git add render.yaml DEPLOY-RENDER.md app/backend/Dockerfile app/backend/.dockerignore \
           "app/backend/src/main/resources/application.yml" "website 2/next.config.js"
   git commit -m "chore: add Render blueprint (backend Docker + Next.js web + Postgres)"
   git push origin main
   ```
2. Go to Render → **New → Blueprint**, pick this repo. Render reads `render.yaml`
   and creates the DB + both services.
3. Fill the `sync: false` env vars (see the two lists below) → **Apply**.
4. After the DB is up, copy its **Internal Database URL** and set the backend's
   `DB_URL` to the JDBC form:
   `jdbc:postgresql://<internal-host>/cropwise`
5. Once the website is live, set the backend's `CORS_ORIGINS` to the website URL
   (e.g. `https://cropwise-web.onrender.com`) and redeploy the backend.

## Option B — create each service manually

- **Postgres**: New → Postgres (free). Note the connection details.
- **Backend**: New → Web Service → this repo → Root Dir `app/backend`,
  Runtime **Docker**. Add the backend env vars. Health check path `/api/schemes`.
- **Website**: New → Web Service → this repo → Root Dir `website 2`,
  Runtime **Node**, Build `npm ci && npm run build`, Start `npm run start`.
  Add the website env vars.

## Env vars

### Backend (`cropwise-backend`)
Set these from your `app/backend/.env`:
```
DB_URL=jdbc:postgresql://<render-internal-host>/cropwise
DB_USERNAME=<from Render Postgres>        # auto-wired by the blueprint
DB_PASSWORD=<from Render Postgres>        # auto-wired by the blueprint
DB_DRIVER=org.postgresql.Driver
JWT_SECRET=<auto-generated by Render>
CORS_ORIGINS=https://<your-website>.onrender.com
GEMINI_API_KEY=...
MARKET_API_KEY=...
SARVAM_API_KEY=...
NEO4J_URI=...
NEO4J_USERNAME=...
NEO4J_PASSWORD=...
```

### Website (`cropwise-web`)
Set these from your `website 2/_env.local` (all the keys already in that file):
```
NEXT_PUBLIC_FIREBASE_API_KEY, NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN,
NEXT_PUBLIC_FIREBASE_PROJECT_ID, NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET,
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID, NEXT_PUBLIC_FIREBASE_APP_ID,
FIREBASE_ADMIN_PROJECT_ID, FIREBASE_ADMIN_CLIENT_EMAIL, FIREBASE_ADMIN_PRIVATE_KEY,
GEMINI_API_KEY, MARKET_API_KEY, SARVAM_API_KEY,
NEO4J_URI, NEO4J_USER, NEO4J_PASSWORD
```
> `FIREBASE_ADMIN_PRIVATE_KEY` spans multiple lines — paste it exactly, keeping the
> `\n` sequences as in `_env.local`.

## Point the Android app at the live backend

In `app/android/app/build.gradle.kts`, change the base URL from the emulator
default to your Render backend and rebuild:
```kotlin
buildConfigField("String", "API_BASE_URL", "\"https://cropwise-backend.onrender.com/\"")
```

## Notes
- Free web services **sleep after ~15 min idle**; the first request then takes
  ~30–60s to wake. Fine for a demo/portfolio.
- The backend's first build runs Maven inside Docker (a few minutes).
- The website build ignores pre-existing TS/lint errors (set in `next.config.js`)
  so it deploys cleanly; fix those incrementally later.
- Never commit real keys — they live only in Render's dashboard and your local
  git-ignored `.env` / `_env.local`.
